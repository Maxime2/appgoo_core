About appGoo

appGoo allows all of your web content to be generated from within a PostgreSQL database via function calls. This allows you to deliver web pages with very complex data manipulation and presentation that is delivered to the end user in milliseconds.

On the right is a list of sample pages that you can click to deliver a new sample page that demonstrates different capabilities of appGoo. Use the browser’s developer tools to view the performance characteristics of each type of content delivery.

All content embedded in a single page

This page serves the entire web page as a single encapsulated web page with all CSS and Javascript embedded into the page itself. The entire page has been served from the database with nothing sourced from the filesystem. This does not mean inline CSS (but of course this can be used), all CSS and Javascript are included as scripts within the page content.

Rough testing reveals that CSS + Javascript < 10kb in size can be efficiently delivered as embedded in the page itself - but there may be other factors to consider before deciding upon this approach. The greatest advantage is that all content is delivered in a single web request. However, http2 may negate this advantage. The most significant disadvantage of this method is that you do not use the browser’s caching. Whilst the performance difference will be minimal, CSS and JS can be either pre-loaded or loaded on the first page and cached so that it is not required to be served again.
