Why use appGoo ?

  • speed
  • scale
  • flexibility
  • simplicity
  • low cost infrastructure

Serving web content from the database

Wordpress and other CMS's already use the power of the database to drive web page content. appGoo is different in the fact that it delivers the raw capability for you to scaffold on your own logic of generating content from within the database. You do not inherit all the overhead and potential bloat of a full-blown CMS, you can choose one of appGoo's application development components, or develop your own to meet your specific requirements.


Harnessing the speed of the database

The PostgreSQL database does not require much tuning until the usage requirements become significant. Therefore it is easy to achieve delivery of complex web pages in milliseconds without the need for specialist tuning skills.

PostgreSQL is fast at serving data. Really fast. Users love speed. The webpage can be static - like this page - but appGoo really shines when significant data manipulation and manipulation of the page content needs to occur. As all processing occurs within the database itself, it is ACID and it is highly efficient. appGoo is primarily designed for applications as opposed to web sites - but excels at web sites too. appGoo loves data at speed, and that is what applications require. Data inserts, updates and deletes can all occur as part of a single ACID transaction that includes the generation of the resulting web page. Analytics with realtime data aggregation and presentation through modern javascript libraries like D3.js are also an ideal use of appGoo that gives users snappy pages that encourages deep and engaging use.


Ease of scale with the database

Engineers have put considerable thought and sophistication into the design of databases - PostgreSQL is no exception. The database that serves web content will naturally scale with the usage of your application - without you having to change anything - until the requirements and usage become significant. This allows you to concentrate on the design of your web site or application.

When massive scale is required you can utilise the standard features of PostgreSQL - replication & clustering as well as add-ons that you can deploy for sharding and other massive performance capabilities.


Transaction integration cross different data types

Seperate logic for accessing data or altering data is not required - your application code is already in the database and working with data is both simple and immediately available. The power of PostgreSQL is that you can work with a mixture of relational data and NoSQL/document structure data through cursors, recordsets or JSON. PostgreSQL's SQL & JSON capabilities include high performing aggregation functions, full text search, parallel queries across multiple CPUs, Common Table Expressions (CTE's), planned queries in memory, recursive queries & the powerful window function calls.

Refer to the PostgreSQL documentation for further details. The result is that you have a high powered data workhorse to serve the needs of your application allowing you to efficiently build, prototype & deliver powerful applications.


API's too

Its not just serving up web pages to users that appGoo can be used for. API requests can also be processed through appGoo and data either inserted, updated, deleted and/or retrieved as a result of the call. appGoo can cater for thousands of APIs all the while serving pages to users. Being an ACID compliant database, the success of the API call will be referentially enforced by the successful completion of the function call within the database. Less complexity for you to cater for in your code, and high performing APIs as a result.


All content in one place

The application code and the related data all co-exist within the database - backups, replication & high availability clusters ensure that both the data and application are recoverable. Both the application version and the data will be in sync due to this co-existence, therefore the complexity of incremental code updates and corresponding data updates is simplified.

The end result is simplicity - allowing you to concentrate on designing & developing your application.


Simple and low cost infrastructure

As web requests can be completed within milliseconds and concurrent scale is a core capability of the database - hundreds and thousands of requests can be served in a second. Whilst it depends upon the nature of the web site or application, it is possible to support thousands of users through an AWS T2.small (1 vCPU, 2GB RAM) at about US$0.02 per hour or less than US$15 per month. Even less for development instances which only need a T2.micro

This is low cost computing. Even larger scale applications with realtime aggregation can support tens or hundreds of users with 4 CPUs and 8GB RAM. The larger the ram, the larger the database cache that is possible to achieve even faster processing where possible.

appGoo can be deployed on the same server as the database resulting in the database being locked away from the outside world and Apache hardened to only permit genuine requests - thus resulting in an extremely simple cost effective architecture that is highly secure and easy to establish and maintain. Focus on the application design and build, not the peripheral deployment challenges of other development architectures.
